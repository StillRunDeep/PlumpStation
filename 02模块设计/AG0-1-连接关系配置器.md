# AG0-1 连接关系配置器

> 功能分类：**用户交互**
> 管道对象命名与阀件定义见 [00-计算定义总纲.md](./00-计算定义总纲.md)

---

## 一、职责概述

AG0-1 提供一个**可交互的流程图编辑器**，让用户定义泵站内所有设备和管道的**拓扑连接关系**与**房间归属**（逻辑层，不含精确空间坐标）。

系统根据 AG0-0 输入的泵数 N 自动生成默认连接拓扑，用户可在此基础上手动调整：增删设备/阀件、编辑流程连线、拖拽设备更换房间归属。确认后输出的拓扑数据供 AG3-1 用于图纸绘制。

---

## 二、在系统中的位置

```
AG0-0（用户对话）→ N（泵数）
  ▼
AG0-1（连接关系配置器）← 用户交互编辑
  │  输出：Topology（节点 + 边 + 房间归属）
  ▼
AG3-1（设备流线二维示意）— 按房间归属渲染设备位置
```

---

## 三、默认拓扑结构

系统自动生成以下默认连接（以 N = 3 为例，含 1 台备用泵）：

```
进水
  ├──► P1 ──► 止1 ──► 闸1 ──┐
  ├──► P2 ──► 止2 ──► 闸2 ──┤──► 出水
  ├──► P3 ──► 止3 ──► 闸3 ──┘
  └──► 备  ──► 止备 ──► 闸备 ─┘
```

所有设备默认归属**泵房维护间**，进水/出水节点为固定锚点（不可删除）。

---

## 四、房间定义

| 房间 ID | 名称 | 说明 |
|---------|------|------|
| `wet_well` | 集水坑 | 显示为蓝色虚线框，左侧 |
| `pump_room` | 泵房维护间 | 显示为浅蓝实线框，右侧 |

设备可通过拖拽跨房间移动；AG3-1 根据设备的 `roomId` 决定绘制位置。

---

## 五、数据模型

### 5.1 节点（Device）

```ts
interface Device {
  id:      string                                    // 唯一标识，如 "pump_1"
  type:    'pump' | 'check_valve' | 'gate_valve'    // 设备类型
  label:   string                                    // 显示名，如 "P1"、"止1"
  roomId:  string                                    // 所属房间 ID
  editorX: number                                    // 编辑器画布 X 坐标
  editorY: number                                    // 编辑器画布 Y 坐标
  isSpare: boolean                                   // 是否备用泵
}
```

**固定节点**（不可删除、不可移动）：

| id | 类型 | 说明 |
|----|------|------|
| `source` | source | 进水锚点（集水坑内） |
| `discharge` | discharge | 出水锚点（画布右侧） |

### 5.2 边（Edge）

```ts
interface Edge {
  id:     string   // 唯一标识
  fromId: string   // 起点节点 id
  toId:   string   // 终点节点 id
}
```

### 5.3 完整拓扑对象

```ts
interface Topology {
  rooms:   Room[]     // 房间定义（含编辑器坐标）
  devices: Device[]   // 所有可编辑设备
  edges:   Edge[]     // 所有连线
}
```

## 六、用户可操作项

| 操作 | 说明 |
|------|------|
| 拖拽设备 | 在画布内自由移动设备；拖入另一房间时自动更新 `roomId` |
| 增加设备 | Toolbar 点击"＋水泵 / ＋止回阀 / ＋电动闸阀"，新设备放置在泵房内 |
| 删除设备 | 选中后点击"删除选中"或按 `Delete` 键，关联连线同步删除 |
| 连线模式 | 开启后依次点击两个节点即可新建有向连线 |
| 删除连线 | 点击连线选中（变红），再按 `Delete` 键删除 |
| 重置 | 恢复为当前 N 值对应的默认拓扑 |
| 确定拓扑 | 点击"✓ 确定拓扑"后，下次点击"开始计算"时 AG3-1 使用该拓扑渲染 |

---

## 七、AG3-1 同步规则

`roomId` 决定设备在 AG3-1 平面图中的渲染区域：

| roomId | AG3-1 渲染位置 |
|--------|----------------|
| `pump_room` | 泵房矩形内（原有位置逻辑） |
| `wet_well` | 集水池矩形内，均匀排布 |

止回阀与电动闸阀随泵的连线关系在平面图中显示，A-A 剖面图不受拓扑影响。

## 八、实现文件

| 文件 | 说明 |
|------|------|
| `src/agents/ag01-topology.js` | 纯数据模型：generateDefaultTopology、addDevice、removeDevice、addEdge、removeEdge、topologyToAG31Params 等 |

# AG0-1 连接关系配置器

> 功能分类：**用户交互**
> 管道对象命名与阀件定义见 [00-计算定义总纲.md](./00-计算定义总纲.md)

---

## 一、职责概述

AG0-1 定义泵站内所有设备和管道的**拓扑连接关系**（逻辑层，不含空间坐标）。

系统根据 AG2-x 的计算结果自动生成默认连接拓扑，用户可在此基础上手动调整：增删阀件、修改管道连接、拖拽端点更改流程。输出的 `ConnectionTopology` 对象供 AG3-1 用于空间布置和图纸绘制。

---

## 二、在系统中的位置

```
AG2-1（水泵选型）      → P_motor, N
AG2-2（管道尺寸计算）  → DN_pumpIn, DN_pumpOut, DN_mainOutlet
AG0-0（用户对话）      → N（泵数）
  ▼
AG0-1（连接关系配置器）
  │  输出：ConnectionTopology
  ▼
AG3-1（设备流线二维示意）
```

---

## 三、默认拓扑结构

系统自动生成以下默认连接（以 N = 3 为例）：

```
WetWell
  ├── PumpInlet_1 ──► Pump_1 ──► PumpOutlet_1 ──► CheckValve_1 ──► GateValve_1 ──┐
  ├── PumpInlet_2 ──► Pump_2 ──► PumpOutlet_2 ──► CheckValve_2 ──► GateValve_2 ──┤──► MainOutlet ──► 排放点
  └── PumpInlet_3 ──► Pump_3 ──► PumpOutlet_3 ──► CheckValve_3 ──► GateValve_3 ──┘
```

规则说明：
- 每台泵独立吸水管（PumpInlet），不共用
- 每台泵出口依次串联：止回阀（CheckValve）→ 闸阀（GateValve）
- 所有泵出口汇入同一总出水管（MainOutlet）

## 四、数据模型

### 4.1 节点（Node）

每个设备或阀件为一个节点：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 唯一标识，如 `pump_1`、`checkValve_2` |
| `type` | Enum | 见节点类型表 |
| `label` | String | 显示名称，如 "P-1"、"CV-1" |
| `spec` | Object | 规格参数（含 DN、P_motor 等，视类型而定）|

**节点类型：**

| type | 中文名 | 典型规格字段 |
|------|--------|-------------|
| `WetWell` | 污水池 | depth, footprintArea |
| `Pump` | 水泵 | flowRate, head, motorPower |
| `CheckValve` | 止回阀 | DN |
| `GateValve` | 闸阀 | DN |
| `ButterflyValve` | 蝶阀 | DN |
| `MainOutletNode` | 总出水汇合点 | DN_mainOutlet |
| `DischargePoint` | 最终排放点 | — |

---

### 4.2 边（Edge）

每条管道连接为一条边：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | 唯一标识，如 `pipe_pump1_cv1` |
| `from` | String | 起点节点 id |
| `to` | String | 终点节点 id |
| `pipeType` | Enum | `PumpInlet` / `PumpOutlet` / `MainOutlet` |
| `DN` | Int | 公称内径（mm），来自 AG1-3 |

---

### 4.3 ConnectionTopology 对象

```ts
interface ConnectionTopology {
  nodes: Node[]      // 所有节点（设备 + 阀件）
  edges: Edge[]      // 所有管道连接
  pumpCount: number  // 工作泵台数 N
}
```

---

## 五、用户可配置项

用户可在默认拓扑基础上调整以下内容：

| 操作 | 说明 |
|------|------|
| 增加阀件 | 在任意边上插入新节点（如加蝶阀、加电动阀） |
| 删除阀件 | 移除节点，边自动重连前后节点 |
| 替换阀件类型 | 如将 GateValve 替换为 ButterflyValve |
| 拖拽端点 | 更改管道连接关系（改变 edge 的 from/to） |
| 增加旁通管 | 新建绕过某节点的 edge |

**约束**：以下连接关系不允许删除（系统必须满足）：
- WetWell → PumpInlet（每台泵必须有独立进水来源）
- Pump → PumpOutlet（水泵出口必须有出水路径）
- 所有 PumpOutlet 路径最终必须连通到 DischargePoint

---

## 六、输入参数

| 参数名 | 来源 | 说明 |
|--------|------|------|
| N | AG0-0 | 工作泵台数，决定默认节点数量 |
| DN_pumpIn | AG2-2 | 自动填入 PumpInlet 边的 DN |
| DN_pumpOut | AG2-2 | 自动填入 PumpOutlet 边的 DN |
| DN_mainOutlet | AG2-2 | 自动填入 MainOutlet 边的 DN |
| P_motor | AG2-1 | 填入 Pump 节点的 motorPower 规格 |

---

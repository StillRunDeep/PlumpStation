# AG4-1 建筑平面布局生成

> 模块版本：v2.0（含空间临近关系与交通组织）

---

## 一、模块目标

在 AG1-2（泵房尺寸）和 AG2-1（水池深度）结果确定后，自动生成 2～3 种完整建筑平面布局方案，供用户选择。每种方案须满足：

- 所有功能空间正确放置在对应楼层
- 有外部直接进入需求的房间必须临近外墙
- 起重机覆盖范围约束满足
- **功能关联房间通过走廊或大厅相互连通**

## 四、交通组织（走廊/大厅）

### 4.1 地面层

| 空间           | 说明                                         |
| ------------ | ------------------------------------------ |
| 停车间+ 泵房维护间   | 核心交通空间，15t 桥式起重机覆盖全区。设备经由卡车停泊→送货口1→泵室竖向进出。 |
| 小型服务房（水表、消防） | 沿外墙排列，各自独立对外门，无需内廊。                        |
| 变压器房         | 直接对外设检修门，无内部流通需求。                          |

> 地面层各房间均有外门，**不需要专用走廊**。维修大厅本身即为主要交通空间。

### 4.2 一层

一层设 **主走廊 `corridor_l1`**（最小净宽 1500 mm），连通以下房间：

```

                  ↕
             [雨水回收房]
```

走廊位置：位于低压配电室与清洁泵/雨水回收房之间，同时与上方风机房区域分隔。

| 参数   | 数值                  |
| ---- | ------------------- |
| 最小净宽 | 1500 mm             |
| 典型净高 | ≥ 2400 mm           |
| 典型位置 | 低压配电室西/东侧，纵向贯通一层南半部 |

## 五、门的放置规则

### 5.1 连接原则

房间之间能否设门，取决于是否存在连接需求：

| 情形                               | 是否可开门  |
| -------------------------------- | ------ |
| 两房间存在 MUST 或 SHOULD 临近关系         | ✓ 可开内门 |
| 房间与交通空间（`corridor_l1` 或维修大厅）相邻   | ✓ 可开内门 |
| 地面层房间有 `ext_access` 约束，接触外墙（非南墙） | ✓ 可开外门 |
| 两房间无任何连接规则关系                     | ✗ 不得开门 |

### 5.2 门的位置

- **内门**：在两房间共用边线（shared wall segment）的重合段内，将门放置在靠近端部的位置（取重合段长度的 1/4 处），以**远离**两房间的各自中心点。
- **外门**：在房间接触外墙的墙段内，同样靠近墙段端部放置。
- 标准门宽：900 mm（单扇）。

### 5.3 特殊情况

- **一层**：普通房间不直接对外墙开门，须经走廊 `corridor_l1` 进出。
- **风机房** `fan_room`：体量大，可在北墙/西墙设检修门，需与走廊相连。
- **吊装口** `dock1` / `dock2`：为楼板开口，不设门。

## 六、平面布局方案逻辑描述

### 基于**约束生长法**（见 `AG4-1_1-约束生长法.md`）对各功能房间进行随机初始位置生长，最终输出多个方案。

#### A.0 用户交互参数

在运行前向用户展示以下可调参数，用户确认或修改后方可进入生长算法。

##### A.0.1 建筑整体尺寸

| 参数               | 代码变量  | 初始值    | 说明              |
| ---------------- | ------ | ------ | --------------- |
| 建筑东西向宽度 BW（mm）  | `bW`   | 18600  | 用户可修改；修改后重新计算网格 |
| 建筑南北向进深 BD（mm）  | `bD`   | 24000  | 用户可修改；修改后重新计算网格 |

> 坐标系：(0,0) 在建筑内侧西北角，X→东，Y→南。  
> 图纸按照上北下南进行方位对应。  
> 用户输入的 BW、BD 作为主方案组（9 个方案）的建筑尺寸。

##### A.0.2 功能空间目标面积

以下为各功能空间目标面积的初始值，用户可逐项修改。标注 `[继承]` 的参数值来自前序模块计算结果，修改前系统给出提示。

**地面层（Ground Floor）：**

标注「★可调」的房间：用户修改面积后，生成器会以该值为目标调整对应尺寸并加入少量随机抖动；其余行为参考尺寸（由建筑比例推算，不受面积输入影响）。

| 空间      | 标识            | 参考面积（m²） | 可调  | 来源说明                          |
| ------- | ------------- | -------- | --- | ----------------------------- |
| 变压器房 1  | `trafo1`      | ≈ 69     | —   | 按 (BW−起重机区宽) × 0.333·BD 推算  |
| 变压器房 2  | `trafo2`      | ≈ 69     | —   | 同上                            |
| **停车区** | `parking`     | ≈ 172    | ★   | 起重机区宽 × 南段深度；默认约 10×17.2 m   |
| 泵房维护间   | `repair_zone` | ≈ 34     | ★   | 起重机区宽 × 北段深度（BD 余量）          |
| 总水表房    | `meter_main`  | ≈ 10     | —   | 按比例推算                         |
| 水表房     | `meter_sub`   | ≈ 9      | —   | 按比例推算                         |
| 消防设备房   | `fire_equip`  | ≈ 7      | —   | 按比例推算                         |

**一层（First Floor）：**

| 空间            | 标识            | 参考面积（m²） | 可调  | 来源说明                       |
| ------------- | ------------- | -------- | --- | -------------------------- |
| 低压配电及 PLC 控制室 | `lv_control`  | ≈ 141    | ★   | 宽度在 7500–9000 mm 间随机       |
| 风机房           | `fan_room`    | ≈ 160    | ★   | 宽按比例，深按用户面积调整（±8 % 随机）     |
| 清洁泵房          | `clean_pump`  | ≈ 40     | ★   | 由 Y 向分割线推算                 |
| 雨水回用泵房        | `rainwater`   | ≈ 57     | ★   | 由 Y 向分割线余量推算               |
| 走廊            | `corridor_l1` | ≈ 20     | —   | 净宽固定 1600 mm，长度由楼层决定，不可单独调整 |

> **提示规则**：若某面积值来自 AG1-2 或 AG2-1 模块的计算输出（标注 `[继承]`），系统在界面中显示：  
> _"此值由前序模块 [模块名] 计算得出，修改后将不再与上游结果保持同步，确认修改？"_  
> 用户确认后仍可修改，修改结果仅影响当前布局生成，不回写上游模块。
> 功能房价的面积虽然有目标，但是横向还是纵向生长不做特殊规定。

##### A.0.3 停车区作为独立功能空间

`parking`（停车区）在起重机覆盖区（`crane15`）内与 `repair_zone` 上下分割：

- 默认参考面积约 172 m²（起重机区宽 10 m × 17.2 m 深）；用户可指定目标面积，生成器以该值推算南北分割线并附加 ±5 % 随机抖动；
- 约束：须完全在 15t 桥式起重机覆盖范围（`crane15` 区域）内；
- 与 `repair_zone` 保持 SHOULD 邻接关系（便于设备转运）；
- 随机决策：`parkingNorth` 标志控制停车区在北（邻近服务房）还是在南（默认），确保方案间布局差异明显。

#### A.1 初始条件（默认值，可通过 A.0 用户交互修改）

| 条件     | 默认数值              | 说明                        |
| ------ | ----------------- | ------------------------- |
| 建筑平面尺寸 | BW 18600 × BD 24000 mm | 用户可在 A.0.1 修改；BW 东西，BD 南北 |
| 网格单元尺寸 | 100 mm            | 网格数 = (BW/100) × (BD/100) |

#### A.2 可变元素（参与生长优化）

随机性来自**种子位置**，不来自面积目标抖动。每个功能空间（或竞争同一区域的两个空间）获得一个随机种子坐标，生长结果由此产生；面积可能与目标不符，这是允许的。

**地面层：**

| 随机维度            | 描述                                                         |
| ----------------- | ----------------------------------------------------------- |
| `trafoEast`       | 变压器区东置或西置（影响所有左右关系）                           |
| `svcOrder`        | 服务用房 3 间的左右顺序（6 种排列）                              |
| `craneZoneSplitEW`| 起重机区：30 % 概率东西分割，70 % 概率南北分割（两种拓扑结构）    |
| `repairSeedRatio` | `repair_zone` 种子位置（在起重机区内随机）                       |
| `parkingSeedRatio`| `parking` 种子位置（与 repairSeed 共同确定分割线和房间归属）     |
| `trafoSeedRatio`  | 变压器区块的随机 Y 起点（变压器组不总是贴北墙）                   |
| `dock1SeedX/Y`    | 吊装口1 在起重机区内的随机坐标（x、y 均独立随机）                 |

> 分割规则：两个种子的中点为分割线；种子靠近北/西端的房间占北/西侧空间——无需另设对调标志。

**一层：**

| 随机维度            | 描述                                                         |
| ----------------- | ----------------------------------------------------------- |
| `lvWRatio`        | 低压配电室宽度在 [7500, 9000] mm 范围内随机（不由面积推算）      |
| `fanDRatio`       | 风机房深度为 BD × [35 %, 60 %] 随机（决定一层北侧区域高度）     |
| `cpSeedRatio`     | 清洁泵房在泵区内的随机种子位置                                   |
| `rwSeedRatio`     | 雨水房在泵区内的随机种子位置（与 cpSeed 确定分割线和南北归属）     |
| `dock2XRatio`     | 吊装口2 在风机房内的随机 x 偏移                                 |

#### A.5 算法流程（每次随机初始化）

```
输入：用户确认的建筑尺寸 BW×BD、各功能空间目标面积（可选）、约束声明
输出：一个完整的地面层 + 一层平面布局

Step 0  用户交互确认（仅首次运行时执行，后续重复生长复用参数）
        展示 A.0 中的所有可调参数及其初始值；
        对继承自上游模块的参数给出提示；
        等待用户确认或修改，锁定 BW、BD 及各房间目标面积。

Step 1  栅格化
        将 BW×BD 建筑轮廓映射为 (BW/100)×(BD/100) 网格；
        按比例标记 crane15、crane5 区域边界。

Step 2  种子位置随机化（13 个独立随机维度，均为位置坐标，不由面积推算）
        ① trafoEast：变压器区东置或西置
        ② svcOrder：服务用房左右排列顺序（3! = 6 种）
        ③ craneZoneSplitEW：起重机区东西分割（30 %）或南北分割（70 %）
        ④ repairSeedRatio：repair_zone 在起重机区内的种子位置
        ⑤ parkingSeedRatio：parking 的种子位置
           （④⑤ 中点为分割线，种子靠北/西者得对应侧空间，无对调标志）
        ⑥ trafoSeedRatio：变压器区块 Y 起点随机偏移
        ⑦ dock1SeedX：吊装口1 x 随机种子
        ⑧ dock1SeedY：吊装口1 y 随机种子（新增）
        ⑨ lvWRatio：低压配电室宽度随机（范围内，不依赖面积）
        ⑩ fanDRatio：风机房深度随机（BD × 35 %~60 %）
        ⑪ cpSeedRatio：清洁泵房在泵区内的种子位置
        ⑫ rwSeedRatio：雨水房的种子位置（⑪⑫ 共同确定分割线和南北归属）
        ⑬ dock2XRatio：吊装口2 x 偏移

Step 3  权重网格生成
        - 有外墙约束的房间：外墙 100mm 内权重 = 10，其余权重 = 1
        - 有邻接约束的房间：已占用邻居周围 200mm 内权重提升 × 3
        - 起重机覆盖区域以外的格子：crane 约束房间权重置 0

Step 4  房间放置（Room Placement）
        按面积比例（大房间优先）逐一选取加权随机初始位置；
        parking 与其他功能空间同等参与竞争；
        若邻接约束未被满足，则重置重试（最多 20 次）。

Step 5  房间扩展（Room Expansion）
        阶段 1 — GrowRect：向最大可用矩形方向扩展，
                            达到用户设定的目标面积后停止；
        阶段 2 — GrowLShape：允许 L 形填满剩余空间；
        阶段 3 — FillGaps：将孤立空格分配给相邻最大房间。

Step 6  连通性处理（Room Connectivity）
        ① 显式连通约束（MUST 邻接对）优先放门；
        ② 走廊 corridor_l1 连接所有相邻房间；
        ③ 可达性检验：若有房间孤立，沿邻接路径补门。
        门位置：共用墙段的 1/4 处（远离房间中心，见 Section 五）。

Step 7  评分
        调用 AG4-2 评分函数，返回 score、spaceEfficiency、breakdown。
```

#### A.6 多方案生成

- 提供生成按钮，每次点击，获得多组方案；

- 生成按钮文字变为筛选，点击筛选，不符合筛选规则的方案被抛弃，符合要求的方案进入候选列表

- 候选列表中的方案按得分高低实时排序。

**筛选规则：**

```
每组内：
  优先保留 violations = 0 的方案，按 score 降序排列；
  若有效方案不足目标数量，补充得分最高的无效方案（注明约束未满足）。
```

## 八、评分项说明

> 评分由 **AG4-2 模块**执行，此处为参考说明。详细评价指标定义见 `AG4-3-建筑平面布局评价.md`。

| 项目        | 分值       | 说明                   |
| --------- | -------- | -------------------- |
| 基础分       | 10000    | —                    |
| 占地面积      | −8 pt/m² | 紧凑方案得分更高             |
| 必须临近满足    | +40/对    | 每满足一对 MUST 关系        |
| 建议临近满足    | +15/对    | 每满足一对 SHOULD 关系      |
| 走廊交通完整    | +20      | 走廊同时与 ≥2 个目标房间临近     |
| 变压器外墙     | +20/台    | 每台变压器接触东/西外墙         |
| 变压器同侧     | +20      | 两台变压器同侧布置            |
| 风机房近送货口   | 最高+30    | 距离线性衰减               |
| **空间有效率** | 最高+50    | 功能房间面积/包围盒面积，见 AG4-2 |
| 约束违反      | −50/项    | 每项必须约束未满足            |

## 九、输入输出定义

**输入**：

```ts
interface UserParams {
  buildingLength: number   // 用户确认的建筑长度 L（mm），默认 24000
  buildingWidth: number    // 用户确认的建筑宽度 W（mm），默认 18600
  roomTargetAreas: {       // 各功能空间目标面积（m²），用户可修改
    trafo1: number         // 默认 20
    trafo2: number         // 默认 20
    parking: number        // 默认 80，独立功能空间参与生长
    repair_zone: number    // 默认 60
    meter_main: number     // 默认 12
    meter_sub: number      // 默认 8
    fire_equip: number     // 默认 15
    lv_control: number     // 默认 65
    fan_room: number       // 默认 55
    clean_pump: number     // 默认 25
    rainwater: number      // 默认 25
    corridor_l1: number    // 默认 18
  }
}
```

**输出**：

```ts
interface LayoutVariant {
  id: string               // 方案编号（A01～A23）
  group: 'user' | 'ratio' // 组别：user = 组 1，ratio = 组 2
  aspectRatio?: number     // 组 2 专属：长宽比（1.0/1.2/1.5/1.8/2.0/2.4）
  buildingSize: { L: number; W: number }  // 实际使用的建筑尺寸（mm）
  template: Template       // 模板定义（A/B/C）
  placements: Placements  // 所有房间坐标 (mm)
  groundPlacements: Placements
  level1Placements: Placements
  violations: Violation[] // 约束违反列表
  adjacency: {
    satisfied: AdjPair[]  // 已满足的临近对
    violated: AdjPair[]   // 未满足的临近对
  }
  feasible: boolean       // violations.length === 0
  score: number           // 综合评分
}
```

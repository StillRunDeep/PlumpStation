# AG4-3 建筑平面布局评价

> 模块版本：v1.0

---

## 一、模块目标

接收 AG4-2 生成的多个布局方案，对每个方案进行量化评价，输出包含**综合评分**和**空间有效率**的评价结果，供用户选择最优方案。

---

## 二、在系统中的位置

```
AG4-2（建筑平面布局生成）
  │  输出：LayoutVariant[]（含 placements, violations, adjacency）
  ▼
AG4-3（建筑平面布局评价）
  │  输出：EvaluatedVariant[]（含 score, spaceEfficiency, breakdown）
  ▼
UI 展示（方案对比与选择）
```

---

## 三、评价指标

### 3.1 综合评分（score）

| 项目 | 分值 | 说明 |
|------|------|------|
| 基础分 | 10000 | 所有方案起始分数 |
| 占地面积 | −8 pt/m² | 包围盒面积越小得分越高，鼓励紧凑布局 |
| 必须临近满足 | +40/对 | 每满足一对 MUST 邻接关系 |
| 建议临近满足 | +15/对 | 每满足一对 SHOULD 邻接关系 |
| 走廊交通完整 | +20 | 走廊同时与 ≥2 个目标房间临近 |
| 变压器外墙 | +20/台 | 每台变压器接触东/西外墙 |
| 变压器同侧 | +20 | 两台变压器布置于同侧 |
| 风机房近送货口 | 最高 +30 | 距离越近得分越高（线性衰减） |
| **空间有效率** | 最高 +50 | 见 3.2，线性映射 |
| 约束违反 | −50/项 | 每项 MUST 约束未满足 |

### 3.2 空间有效率（space_efficiency）

```
space_efficiency = Σ(功能房间面积) / (buildingW × buildingD)
```

**功能房间**：所有已放置房间，**除外**：
- 走廊 `corridor_l1`（交通空间）
- 吊装口 `dock1`、`dock2`（楼板开口，非封闭房间）

**含义**：走廊或空白区域越多，有效率越低。

**评分映射（线性）**：

| 空间有效率 | 附加分 |
|-----------|--------|
| ≥ 0.90 | +50 |
| 0.80 | +33 |
| 0.70 | +17 |
| ≤ 0.60 | 0 |

公式：`efficiencyScore = clamp((efficiency - 0.60) / 0.30, 0, 1) × 50`

## 四、输入参数

| 参数 | 类型 | 来源 | 说明 |
|------|------|------|------|
| variants | LayoutVariant[] | AG4-2 | 所有布局方案（含坐标、约束、邻接信息） |

---

## 五、输出参数

```ts
interface EvaluatedVariant extends LayoutVariant {
  score: number              // 综合评分
  spaceEfficiency: number    // 0~1，功能房间面积占比
  efficiencyScore: number    // 空间有效率对应的附加评分（0~50）
  scoreBreakdown: {          // 各项得分明细
    base: number
    footprint: number
    adjacency: number
    corridor: number
    trafo: number
    fanRoom: number
    efficiency: number
    violations: number
  }
}
```

方案按 `score` 降序排列，最优方案排第一。

---

## 六、排除空间说明

以下空间不计入空间有效率分子（功能面积）：

| 空间 ID | 原因 |
|---------|------|
| `corridor_l1` | 走廊为交通空间，非功能房间 |
| `dock1` | 楼板吊装口，无围合面积 |
| `dock2` | 楼板吊装口，无围合面积 |

所有其他已放置房间均计入功能面积。
